Linear Gradient - The Answer To 'Can We Make It Pop?'

Need something a touch smoother or extreme contrast? Linear Gradient adds a little pop right where the client really needs it.

But What Is Linear Gradient?

Linear gradients provide an element with multiple colors. This is accomplished by switching the angle along the gradient line. The gradient line is a bit much but just in case click on the ? below:

MDN Linear-Gradient

The gradient line goes through the center of a colored image at different angles. Changing the angle is what changes the color we see on the page. The lines starting point and ending point are symmetricly placed around the center of the colored image where it creates an angle with the closest corner. Adding stopping points will create more color transitions. Stopping points will be halfway between the nearest two points. However you could also define length and percentage in the CSS data type.

By adding additional stopping points along the gradient line there will be more colors to the element. 

element.class {

	background: linear-gradient( direction, firstColor, secondColor );

}

Direction can be an angle or a direction for the colors to flow. If geometry isn't your thing go ahead and use `to top` or `to top left`. These two options will render the same result,

element.class {

	background: linear-gradient( to top, firstColor, secondColor );
		/*is the same result as
	background: linear-gradient( 0deg, firstColor, secondColor );
	

}

also remember that you can add more colors by adding more stops

element.class {

	background: linear-gradient( to top, firstColor, secondColor, thirdColor, fourthColor, etc. );
		
}

Notice how fancy the text was at the top and wondering why linear-gradient isn't working on text? Linear-gradient can only be applied to an element that can use an image. For example, an image can be used as the background of a div so linear-gradient can be applied. If you really need that early power-point style there is a workaround. Make the background of the text element linear-gradient and turn the text transparent. 

element {
  background: linear-gradient(direction, firstColor, secondColor);
  -webkit-background-clip: text;
  -webkit-text-fill-color: transparent;
}

Now the text will be transparent and you will see the background as the linear-gradient.

So what about those fancy borders on the sides, linear-gradient doesn't work on the border either? The border needs a little trickery to accept linear-gradient. 

element {
	border-image: [source] [slice] [width];
}

border-image can set the background image of a border. Now that you have access to the border as an image, linear-gradient away. [slice] is the slice where the image will be cut and can be a number or a percent. There is no need to cut the linear-gradient for the border so we use 1 for slice.

element {
	border-image: linear-gradient(direction, firstColor, secondColor) 1 100%;
}

While calling colors by name such as red, orange, yellow - there is a sharper change than using colors that are similar. Check out the color wheel of development for some hex codes and grab something closer together if you need a smooth finish. Greys work well for very sublte changes.

You want to loop the gradient colors? well then, repeating-linear-gradient here we come.
