What is an Object in Javascript

Lets start with what is not an object in javascript: strings ("Tennis Ball"), numbers (42), true, false, null, and undefined. Everything else will be an object.

An object is a variable which can contain other variables. Think of an object like a container that can store any other type of data or even another container full of data. The contained variables are called named values which are simply written name : value

var myObj = {
  name: value,
  name2: value2,
  innerObj: { name3: value3,
              name4: value4
            }
}

Objects can store any number of name : value pairs which is very useful for holding a group of data that is all related in some way. for example if you had a contact list this might be one of the contacts

var Contact = {
  firstName: "Timber",
  lastName: "Underfoot",
  phone: 9783450098,
  email: "TimberTheWolf@barkmail.com"
}

An object's named values are called properties of that object. Consider that the Contact object above has the information regarding a Timber Underfoot - the properties of the object are firstName, lastName, phone, and email. Each of those properties are assigned a value, hence name : value.

How to create an Object

There are many ways to create an object. Depending on how you plan to use the object will dictate which option you use.

Object Literal

Object Literal is the most straight forward way to create an object

var Contact = {
  firstName: "Timber",
  lastName: "Underfoot",
  phone: 9783450098,
  email: "TimberTheWolf@barkmail.com"
}

The Contact object is a variable which uses curly brackets {} and then name:value pairs seperated by commas. The object could be placed all on one line, however, readability is important so common practice is to have each name:value pair on a new line.

Keywork New

This method is rarely used and hurts readability in this developer's opinion. If you were interested in usinging the keywork new here is how an object is setup.

var Contact = new Object();
  Contact.firstName = "Timber";
  Contact.lastName = "Underfoot";
  Contact.phone = 9783450098;
  Contact.email = "TimberTheWolf@barkmail.com";

Although this way is not typical it does show a great example of what is going on with an object. In the above example we named our variable Contact and then assigned it to new Object(); At this point the object is empty. By accessing the object and adding a property value with an assignment you will start to build the object. 

var Contact = new Object();
  console.log(Contact);
    // Would return an empty object
    // {}
    
  Contact.firstName = "Timber";
  Contact.lastName = "Underfoot";
  console.log(Contact)
    //Now it would return the object with both of the above properties added
    // {firstName: "Timber", lastName: "Underfoot"}
  Contact.phone = 9783450098;
  Contact.email = "TimberTheWolf@barkmail.com";

If you are sure you are only going to be creating one object and you have all the information you need the object literal method will be the best option for you. What if you do not know how many properties the object will have? What if you need a whole bunch of contacts not just one? You will need to use an object constructor.

Object Constructor 

function Contact(first, last, phone, email){
   this.firstName = first;
   this.lastName = last;
   this.phone = phone;
   this.email = email;
}
var timber = new Contact("Timber", "Underfoot", 9783450098,"TimberTheWolf@barkmail.com");
var charlie = new Contact("Charlie", "Deephole", 9781789142,"CharlieChompers@barkmail.com");

The above creates an object "Contact". This Contact object is now the prototype.OWe then define the properties we would like with this.propertyName. In this instance the "this" keyword is referring to the function Contact(); which will translate to the new objects when they are created. Assigning a variable to the function using the new keyword here will create the same object with the arguments as properties. This can be done as many times as needed which makes this method perfect for a dynamic contact list so you can add new contacts easily with just one line of code instead of making multiple object literals.

!Important Note

The objects you create are mutable. This means that if you assign an object to a variable such as: var timberContact = Contact; the variable timberContact now controls the entirey of the Contact object. If you were to then add timberContact.firstName = "Lassie"; Lassie would be assigned to all objects that were created using that constructor.

Accessing the Object

With everything stored in one place for your contacts it will be easy to get the information. To access the value of the properties directly you will need to know the property name. 

objectName.property
  // e.g. timber.lastName
objectName["property"]
  // e.g. timber["lastName"]

This method works well when you know the name of the property that you need to access. If you wanted to print out all information of the contact without having to add 4 different lines of code you could use a for...in loop to preform an action on each property in an object. The way a for...in loops work is that for each iteration, one property from the object is assigned to a variable and then a block of code is excuted. 

for (prop in object){
  //code block
}

To confirm - prop above can be labelled anything you like. During the loop the first property in object will be assigned to prop which can then be used in the code block followed by the second property of object and so on. This would be helpful if you needed to print out all information from an object for example

var info;
for( prop in timber){
  info += timber[prop] + ' ';
}
console.log(info);
// "Timber Underfoot 9783450098 TimberTheWolf@barkmail.com "

Now as you can see this is a very interesting loop since you can add any type of function within that code block to be executed on each element. Now that we can go through all the properties we should know how to add or remove these properties. For our Contact object lets say we would like to add an address property

//it is assumed our Contact object is already created and we have made timber a Contact object
timber.address = "42 Galaxy Road, Earth";

//timber would become: 
timber = {
  firstName: "Timber",
  lastName: "Underfoot",
  phone: 9783450098,
  email: "TimberTheWolf@barkmail.com",
  address: "42 Galaxy Road, Earth"
}
//Contact would not be changed

There is now a property address within the timber object. Something interesting to note is that if you were be using the object constructor you would be able to add address to ALL objects that are created by that constructor. This is because you are changing the prototype of the object. For the example with Timber and Charlie above, If you only added timber.address = "42 Galaxy Road, Earth"; then Charlie's contact would not have been updated at all and new contact objects would not have the additional property included. That means the timber object will have 5 properties while the Contact and Charlie objects only have 4. However, if you want address to be available for all current and future contact list items from this example you could add the following:

 Contact.address = "";

This would add the address property to all objects that were construted using the Contact object. If you added a property that you no longer need it is very simple to remove it from an object

delete Contact.address;

That is it... Almost. If you try to delete a property on an object created from the prototype object you will find that you cannot delete inherited properties and it must be removed at the highest level.

Creating Object Methods

The pattern is fairly clear when it comes to creating named values and adding a method to an object follows that same pattern: name : function

//it is assumed our Contact object is already created
Contact.fullName = function() {
  return this.firstName + " " + this.lastName;
  }

Now you can call Contact.fullName(); and run that method. Remember - since this will be inherited to all Contact objects you could also do timber.Fullname() to run the method on that specific object.

Using Prototype Objects

Now that you can create, read, update, and delete data within an object and create brand new objects we will go over the best and fastest way to make your objects.

Object.prototype

As mentioned above the Contact object is a constructor function and is the prototype of any contact objects created from it. Both timber and charlie's objects were created from the Contact.prototype. Because of this anything added to the Contact object will be available to the other objects through inheritence. Another neat piece to prototype objects is that they are a reference for all of the information needed for new objects. This means that the new objects do not contain the methods in the prototype, when a method is called on a particular object the prototype is refereced. This is beneficail because it saves space and time when processing your javascript. Lets take a look at that here is the constructor object which will be our prototype: 
function Contact(first, last, phone, email){
   this.firstName = first;
   this.lastName = last;
   this.phone = phone;
   this.email = email;
   this.fullName = function() {
  return this.firstName + " " + this.lastName;
  }
}

No we will create a reference to that object:

  
var timber = new Contact("Timber","Underfoot",9783450098,"TimberTheWolf@barkmail.com");

The variable timber now references the Contact.prototype object. When you call timber.fullName() you are actually accessing Contact.prototype.fullName(); via the reference that was made using the variable timber. What is happening here is that the "this" keyword in the prototype will be replace by timber and bound to the variable timber.Therefore, the variable timber is a reference to the prototype object and has access to all of the object prototype methods. This is handy so you Don't Repeat Yourself(DRY) writing the fullName() method in each iteration of the Contact objects.
